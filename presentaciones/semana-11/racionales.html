Funciones racionales en SymPy

cancel(), fraction(), apart() y el protocolo “Dominio Primero”

import sympy as sp x = sp.Symbol('x') # Definir una función racional P = x**3 - 2*x**2 - 5*x + 6 Q = x**2 - 4 f = P / Q # Factorizar print(sp.factor(P)) # (x-3)(x-1)(x+2) print(sp.factor(Q)) # (x-2)(x+2) # Simplificar (cancela factores comunes) f_simple = sp.cancel(f) print(f_simple) # => (x**2 - 4*x + 3)/(x - 2) # Separar numerador y denominador num, den = sp.fraction(f_simple) # Fracciones parciales print(sp.apart(f, x)) # Clasificar: propia vs impropia grado_P = sp.degree(P, x) # 3 grado_Q = sp.degree(Q, x) # 2 tipo = "propia" if grado_P < grado_Q \ else "impropia" print(tipo) # impropia
Función SymPy Qué hace
sp.factor(expr) Factoriza un polinomio
sp.cancel(f) Cancela factores comunes P/Q
sp.fraction(f) Separa en (numerador, denominador)
sp.apart(f, x) Descompone en fracciones parciales
sp.degree(P, x) Grado del polinomio
sp.div(P, Q, x) División: (cociente, residuo)

Hueco vs. asíntota

cancel() elimina factores comunes. Si (x+2) desapareció al simplificar, hay un hueco en x = −2. Si un factor queda en el denominador simplificado, es asíntota vertical.

Protocolo: siempre factorizar y simplificar con cancel() ANTES de cualquier análisis. Esto clasifica automáticamente huecos vs. asíntotas.

Asíntotas: solveset(), limit() y div()

Encontrar y verificar verticales, horizontales y oblicuas

import sympy as sp x = sp.Symbol('x') f = (2*x**2 + 3*x - 5) / (x**2 - x - 6) # === ASÍNTOTAS VERTICALES === f_simple = sp.cancel(f) _, den = sp.fraction(f_simple) av = sp.solveset(den, x, sp.S.Reals) print("AV:", av) # {-2, 3} # Verificar con límites laterales for a in av: print(f"x->{a}+:", sp.limit(f, x, a, '+')) print(f"x->{a}-:", sp.limit(f, x, a, '-')) # === ASÍNTOTA HORIZONTAL === ah = sp.limit(f, x, sp.oo) print("AH:", ah) # 2 (grados iguales) # === ASÍNTOTA OBLICUA (otro ejemplo) === g = (x**2 + 3*x + 1) / (x - 1) P2 = x**2 + 3*x + 1 Q2 = x - 1 cociente, residuo = sp.div(P2, Q2, x) print("Oblicua:", cociente) # x + 4 # Verificar: f(x) - oblicua -> 0 print(sp.limit(g - cociente, x, sp.oo)) # 0
AV: {-2, 3} x->-2+: -oo x->-2-: +oo x->3+: +oo x->3-: -oo AH: 2 Oblicua: x + 4
Tipo Condición Código
Vertical Ceros del den. simplificado solveset(den, x)
Horizontal Límite en ±∞ limit(f, x, oo)
Oblicua deg(P) = deg(Q) + 1 div(P, Q, x)[0]

Hecho importante

Una función racional puede cruzar su AH. La asíntota describe el comportamiento cuando x → ±∞, no es una barrera.

Graficar funciones racionales en Matplotlib

El truco de np.nan para eliminar líneas espurias

import numpy as np import matplotlib.pyplot as plt # f(x) = (2x+1) / (x² - x - 6) x = np.linspace(-8, 8, 2000) num = 2*x + 1 den = x**2 - x - 6 with np.errstate(divide='ignore', invalid='ignore'): y = num / den # CLAVE: reemplazar valores extremos con NaN y[np.abs(y) > 50] = np.nan # También: NaN cerca de cada AV for av in [3, -2]: y[np.abs(x - av) < 0.05] = np.nan # Graficar fig, ax = plt.subplots(figsize=(10, 7), layout='constrained') ax.plot(x, y, 'b-', lw=2, label='f(x)') # Asíntotas ax.axvline(x=3, color='red', ls='--', alpha=0.7, label='AV: x=3') ax.axvline(x=-2, color='red', ls='--', alpha=0.7, label='AV: x=-2') ax.axhline(y=0, color='green', ls='--', alpha=0.7, label='AH: y=0') ax.set_xlim(-8, 8); ax.set_ylim(-15, 15) ax.legend(); ax.grid(True, alpha=0.3) plt.show()

El problema

Matplotlib conecta puntos consecutivos con líneas rectas. Cerca de una AV, esto genera una línea vertical espuria que no existe en la función real.

La solución: np.nan

Reemplazar con np.nan los valores cercanos a las asíntotas y los valores extremos. nan rompe la línea del gráfico, eliminando el artefacto visual.

Código de colores estándar

  • Curva: azul sólido (lw=2)
  • Asíntota vertical: rojo punteado ('--')
  • Asíntota horizontal: verde punteado
  • Asíntota oblicua: naranja punteado
  • Hueco: círculo blanco con borde azul

Usar siempre errstate(divide='ignore') para evitar warnings de división por cero. NumPy genera inf y nan que luego limpiamos.

Funciones inversas con SymPy

solve() para despejar, simplify() para verificar composición

import sympy as sp x, y = sp.symbols('x y') # f(x) = (2x + 3) / (x - 1) f = (2*x + 3) / (x - 1) # Paso 1: Resolver y = f(x) para x inv_sol = sp.solve(sp.Eq(y, f), x) print("f_inv(y) =", inv_sol) # => [(y + 3)/(y - 2)] # Paso 2: Reescribir con x f_inv = inv_sol[0].subs(y, x) print("f_inv(x) =", f_inv) # => (x + 3)/(x - 2) # Paso 3: Verificar composición comp1 = sp.simplify(f.subs(x, f_inv)) print("f(f_inv(x)) =", comp1) # x comp2 = sp.simplify(f_inv.subs(x, f)) print("f_inv(f(x)) =", comp2) # x # Dominios y rangos print("Dom(f): R \\ {1}") print("Ran(f): R \\ {2}") print("Dom(f_inv): R \\ {2} = Ran(f)")
f_inv(y) = [(y + 3)/(y - 2)] f_inv(x) = (x + 3)/(x - 2) f(f_inv(x)) = x f_inv(f(x)) = x

Método: resolver para x

En vez de “intercambiar x e y”, despejamos x de la ecuación y = f(x). El resultado es la inversa. Luego renombramos y → x.

Propiedad clave

Dom(f&supmin;¹) = Ran(f) y Ran(f&supmin;¹) = Dom(f). Las asíntotas de f y f&supmin;¹ se intercambian: la AV de f se convierte en AH de f&supmin;¹.

En ciencia de datos: scaler.inverse_transform() deshace StandardScaler. En GLMs, la función de enlace inversa mapea predicciones lineales a la escala original.

Logaritmos en Python: NumPy y SymPy

Cálculo numérico vectorizado + manipulación simbólica de propiedades

import numpy as np import sympy as sp # === NumPy: cálculo numérico vectorizado === x = np.array([1, np.e, 10, 100, 1000]) np.log(x) # ln: [0, 1, 2.30, 4.61, 6.91] np.log10(x) # log10: [0, 0.43, 1, 2, 3] np.log2(x) # log2: [0, 1.44, 3.32, 6.64, 9.97] np.log1p(0.001) # ln(1+x), preciso para x≈0 # === SymPy: manipulación simbólica === x, y = sp.symbols('x y', positive=True) # expand_log: aplica propiedades sp.expand_log(sp.log(x * y)) # => log(x) + log(y) sp.expand_log(sp.log(x**3)) # => 3*log(x) # logcombine: operación inversa n = sp.Symbol('n', real=True) sp.logcombine(sp.log(x) + sp.log(y)) # => log(x*y) sp.logcombine(n * sp.log(x)) # => log(x**n) # Cambio de base sp.log(x, 10) # log10(x) sp.log(x, 2) # log2(x)
Función NumPy SymPy

log&sub1;&sub0;(x) np.log10(x) sp.log(x, 10)

ex np.exp(x) sp.exp(x)

positive=True es obligatorio

Sin esta restricción, SymPy no expande log(x²) a 2·log(x) porque no sabe si x > 0. Esto refuerza que log(x²) = 2·log(x) solo vale para x > 0.

En Python, log = ln

Tanto en NumPy como en SymPy, log() sin base es el logaritmo natural (base e). Para base 10 o base 2, usar las funciones específicas.

Composición de funciones en Python

.subs() para composición general — compose() solo para polinomios

import numpy as np import sympy as sp # === Composición numérica (NumPy) === f = lambda x: np.exp(x) g = lambda x: np.log(x) fog = lambda x: f(g(x)) # e^(ln(x)) = x gof = lambda x: g(f(x)) # ln(e^x) = x x_vals = np.array([1, 2, 3, 4, 5]) print(fog(x_vals)) # [1. 2. 3. 4. 5.] print(gof(x_vals)) # [1. 2. 3. 4. 5.] # === Ejemplo NO conmutativo === h = lambda x: x**2 k = lambda x: np.log(x) print("h(k(e)):", h(k(np.e))) # 1.0 print("k(h(e)):", k(h(np.e))) # 2.0 # === Composición simbólica (SymPy) === x = sp.Symbol('x', positive=True) f_s = sp.exp(x) g_s = sp.log(x) # Usar .subs() (NO compose()) fog_s = f_s.subs(x, g_s) print(sp.simplify(fog_s)) # x gof_s = g_s.subs(x, f_s) print(sp.simplify(gof_s)) # x

compose() vs .subs()

sympy.compose() solo funciona con polinomios (módulo sympy.polys). Para composición general con exp, log, trig, la forma correcta es .subs().

No conmutativa

h(k(x)) = (ln(x))² ≠ k(h(x)) = ln(x²) = 2·ln(x).
En x = e: h(k(e)) = 1, pero k(h(e)) = 2.

Composición en ciencia de datos

  • Pipelines de sklearn: cada paso transforma la salida del anterior
  • Redes neuronales: cada capa aplica σ(Wx + b), la red completa es f∘f∘...∘f
  • Regresión exp: log → regresión lineal → exp (para volver a escala original)

La composición es la base de TODO pipeline de datos. Cuando hacen log-transform → fit → exp-transform, están componiendo funciones.

curve_fit, Polynomial.fit y aplicaciones con datos de Costa Rica

Regresión exponencial y logarítmica

curve_fit para ajuste directo + Polynomial.fit para transformación lineal

import numpy as np from scipy.optimize import curve_fit from numpy.polynomial import Polynomial # === Método 1: curve_fit (ajuste directo) === def modelo_exp(x, a, b): return a * np.exp(b * x) def modelo_log(x, a, b): return a + b * np.log(x) # Ajustar exponencial popt, pcov = curve_fit(modelo_exp, x_data, y_data, p0=[1.0, 0.3]) a, b = popt errores = np.sqrt(np.diag(pcov)) # Ajustar logarítmica popt_log, _ = curve_fit(modelo_log, x_pos, y_log) # === Método 2: Polynomial.fit (linealizar) === # ln(y) = ln(a) + b*x → recta ln_y = np.log(y_data) p = Polynomial.fit(x_data, ln_y, deg=1) coefs = p.convert().coef # [ln(a), b] a_est = np.exp(coefs[0]) # === R² === def r2(y_real, y_pred): ss_res = np.sum((y_real - y_pred)**2) ss_tot = np.sum((y_real - y_real.mean())**2) return 1 - ss_res / ss_tot
Método Ventajas Limitaciones
curve_fit Ajuste directo, sin sesgo Necesita p0 (valores iniciales)
Polynomial.fit Rápido, estable Solo si y > 0, introduce sesgo

Polynomial.fit vs polyfit

np.polyfit es legacy desde NumPy 1.4. Usar siempre numpy.polynomial.Polynomial.fit(). Los coeficientes van de grado bajo a alto (opuesto a polyfit).

Comparar modelos con AIC

AIC = n·ln(RSS/n) + 2k. Se elige el modelo con menor AIC. Solo comparar modelos ajustados a la misma variable respuesta.

Para el proyecto: ajusten al menos 2 modelos (exponencial, logarítmico, polinomial) y comparen con R² y AIC. curve_fit es la herramienta más versátil.

Aplicación CR: tipo de cambio y población

Modelos exponenciales y logísticos con datos del BCCR e INEC

import numpy as np from scipy.optimize import curve_fit # === Tipo de cambio USD/CRC (BCCR) === anios = np.array([1975,1985,1995,2005,2015], dtype=float) tc = np.array([8.57, 50.45, 179.73, 477.79, 534.57]) t = anios - 1975 # normalizar def exp_model(t, P0, r): return P0 * np.exp(r * t) popt, _ = curve_fit(exp_model, t, tc, p0=[8, 0.1]) print(f"TC = {popt[0]:.1f}·e^({popt[1]:.4f}·t)") print(f"Depreciación: {popt[1]*100:.1f}% anual") print(f"T. duplicación: {np.log(2)/popt[1]:.1f} años") # === Población CR (INEC) === anios_p = np.array([1950,1970,1990,2011,2022]) pob = np.array([0.869,1.731,3.049,4.302,5.044]) def logistico(t, K, A, r): return K / (1 + A * np.exp(-r * t)) t_p = anios_p - 1950 popt_l, _ = curve_fit(logistico, t_p, pob, p0=[6, 6, 0.05]) print(f"Capacidad de carga: {popt_l[0]:.1f}M")

Tipo de cambio: exponencial pura

De ¢8.57 (1975) a ¢534 (2015): 40 años de depreciación exponencial. Desde 2015 se estabilizó con flotación administrada (~¢465 en 2026).

Población: exponencial vs. logístico

El modelo exponencial sobreestima (predice 15M en 2070). El logístico captura la saturación: la tasa de fertilidad cayó de 6 (1950) a 1.31 hijos/mujer (2021). Capacidad de carga ~5.5–6M.

Conexión con ML: sigmoide y log-loss

La función exponencial como corazón de la clasificación

import numpy as np import matplotlib.pyplot as plt fig, axes = plt.subplots(1, 2, figsize=(13, 5), layout='constrained') # --- Sigmoide --- x = np.linspace(-8, 8, 300) sigmoid = 1 / (1 + np.exp(-x)) ax = axes[0] ax.plot(x, sigmoid, lw=2.5, color='#FF8C00') ax.axhline(y=0.5, color='gray', ls=':') ax.axhline(y=0, color='red', ls='--', alpha=0.3) ax.axhline(y=1, color='red', ls='--', alpha=0.3) ax.set_title(r'$\sigma(x) = \frac{1}{1+e^{-x}}$') ax.set(xlabel='x (logit)', ylabel='σ(x)') # --- Log-loss --- p = np.linspace(0.001, 0.999, 300) ax = axes[1] ax.plot(p, -np.log(p), lw=2.5, color='#FF8C00', label='y=1: $-\\log(p)$') ax.plot(p, -np.log(1-p), lw=2.5, color='#2d2d2d', label='y=0: $-\\log(1-p)$') ax.set(xlabel='Probabilidad predicha', ylabel='Pérdida', ylim=(0, 6)) ax.set_title('Log-loss (entropía cruzada)') ax.legend() plt.show()
Sigmoide

σ(x) = 1/(1+e−x)

Composición de exp, suma, división. Transforma cualquier número real al intervalo (0, 1). Función de activación en regresión logística.

Log-loss

L = −[y·log(p) + (1−y)·log(1−p)]

Penaliza exponencialmente las predicciones confiadas pero incorrectas. Función de costo estándar para clasificación binaria.

Softmax

pi = ezi / Σezj

Generalización de sigmoide para múltiples clases. Transforma un vector de números en probabilidades que suman 1.

¿Por qué importa esto ahora?

Cada vez que un modelo de ML calcula una predicción de probabilidad, está usando composiciones de funciones exponenciales y logarítmicas. El tema de hoy es la base matemática de los cursos de ML que vienen después.

La sigmoide es literalmente una función racional de ex: σ(x) = ex/(ex + 1). Combina todo lo que hemos visto: racionales, exponenciales, composición e inversas.
