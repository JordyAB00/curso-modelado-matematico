Funciones trigonométricas en NumPy

Vectorizadas, rápidas y siempre en radianes

import numpy as np # === Conversiones grados ↔ radianes === np.radians(180) # → 3.14159... np.degrees(np.pi) # → 180.0 # === Funciones directas (vectorizadas) === theta = np.linspace(0, 2*np.pi, 100) seno = np.sin(theta) # 100 valores coseno = np.cos(theta) # simultáneamente tangente = np.tan(theta) # === Funciones inversas === np.arcsin(0.5) # → π/6 np.arccos(0) # → π/2 np.arctan(1) # → π/4 # arctan2(y, x): cuadrante correcto np.arctan2(1, -1) # → 3π/4 (Q2) np.arctan2(-1, -1) # → -3π/4 (Q3) # === Identidad pitagórica (verificación) === x = np.linspace(0, 2*np.pi, 50) np.allclose(np.sin(x)**2 + np.cos(x)**2, 1) # → True

Regla de oro

NumPy siempre trabaja en radianes. np.sin(90) NO da 1 porque 90 se interpreta como 90 radianes. Usar siempre np.sin(np.radians(90)).

Vectorización

Una sola llamada a np.sin(arr) aplica la función a millones de valores en microsegundos. Es entre 50 y 100 veces más rápido que un bucle Python puro.

arctan2 vs arctan

arctan(y/x) pierde información del cuadrante. arctan2(y, x) recibe los dos argumentos por separado y devuelve el ángulo correcto en (−π, π].

Estas funciones son la base de todo lo que viene: codificación cíclica, ajuste de modelos sinusoidales, positional encoding y similaridad coseno.

El truco de np.nan para la tangente

Reemplazamos los valores extremos con np.nan para que matplotlib no dibuje líneas espurias entre los infinitos positivo y negativo cerca de las asíntotas verticales.

Relación fundamental

cos(x) = sin(x + π/2). El coseno es el seno desplazado π/2 hacia la izquierda. Esta relación de fase aparece en la codificación posicional de los transformers.

La forma general: y(x) = A · sin(B(x − C)) + D

Cuatro parámetros que describen cualquier sinusoide

import numpy as np import matplotlib.pyplot as plt def sinusoide(x, A, B, C, D): """Forma general: A·sin(B(x-C)) + D""" return A * np.sin(B * (x - C)) + D x = np.linspace(0, 24, 500) # Demanda eléctrica de Costa Rica (modelo) A = 400 # amplitud (MW) B = np.pi/12 # período de 24 h: T = 2π/B C = 6 # desfase: máximo a las 12 m.d. D = 1200 # línea base (MW) P = sinusoide(x, A, B, C, D) fig, ax = plt.subplots(figsize=(10, 5), layout='constrained') ax.plot(x, P, lw=2.5, color='#FF8C00') ax.axhline(D, ls='--', color='gray', label=f'D = {D} MW') ax.fill_between(x, D-A, D+A, alpha=0.1, color='#FF8C00') ax.set_xlabel('Hora del día') ax.set_ylabel('Demanda (MW)') ax.set_title('Demanda eléctrica de CR (modelo)') ax.legend() ax.grid(True, alpha=0.3) plt.show()
A

Amplitud

(máx − mín) / 2. Distancia desde la línea base hasta el pico. En CR: A = 400 MW.

B

Frecuencia angular

B = 2π/T donde T es el período. Para ciclo diario T=24, B=π/12.

C

Desfase

Cuánto se desplaza horizontalmente. C=6 hace que el máximo ocurra al mediodía.

D

Desplazamiento vertical (línea base)

(máx + mín) / 2. Es el promedio alrededor del cual oscila la señal. En CR: D = 1200 MW (demanda promedio).

Saber calcular A, B, C, D a mano nos da intuición. Cuando ajustamos a datos reales, scipy.optimize.curve_fit los encuentra automáticamente.

Efecto de la amplitud A y la frecuencia B

Cómo cambia la onda al modificar cada parámetro

import numpy as np import matplotlib.pyplot as plt x = np.linspace(0, 4*np.pi, 500) fig, axes = plt.subplots(1, 2, figsize=(14, 5), layout='constrained') # === Efecto de la amplitud A === for A in [0.5, 1, 2, 3]: axes[0].plot(x, A*np.sin(x), lw=2, label=f'A = {A}') axes[0].set_title('Efecto de A: estira/comprime vertical') axes[0].set_xlabel('x') axes[0].set_ylabel('A · sin(x)') axes[0].grid(True, alpha=0.3) axes[0].legend() axes[0].axhline(0, color='k', lw=0.5) # === Efecto de la frecuencia B === for B in [0.5, 1, 2, 4]: T = 2*np.pi/B axes[1].plot(x, np.sin(B*x), lw=2, label=f'B={B}, T={T:.2f}') axes[1].set_title('Efecto de B: cambia el período') axes[1].set_xlabel('x') axes[1].set_ylabel('sin(B · x)') axes[1].grid(True, alpha=0.3) axes[1].legend() plt.show()

Amplitud A

Multiplica todos los valores de la onda por A. La forma es idéntica pero estirada (A > 1) o comprimida (A < 1) verticalmente. Si A es negativo, la onda se refleja respecto al eje x.

Frecuencia angular B

Comprime o estira horizontalmente. B grande → ondas apretadas (período corto). B pequeño → ondas estiradas (período largo). La regla siempre es: T = 2π / B.

Período deseado Valor de B
24 h (ciclo diario) π/12 ≈ 0.262
7 días (semanal) 2π/7 ≈ 0.898
12 meses (anual) π/6 ≈ 0.524
12.4 h (mareas) 2π/12.4 ≈ 0.507

Efecto del desfase C y la línea base D

Dónde ocurre el máximo y alrededor de qué valor oscila

import numpy as np import matplotlib.pyplot as plt x = np.linspace(0, 4*np.pi, 500) fig, axes = plt.subplots(1, 2, figsize=(14, 5), layout='constrained') # === Efecto del desfase C === for C in [0, np.pi/4, np.pi/2, np.pi]: axes[0].plot(x, np.sin(x - C), lw=2, label=f'C = {C:.2f}') axes[0].set_title('Efecto de C: desplaza horizontal') axes[0].set_xlabel('x') axes[0].set_ylabel('sin(x - C)') axes[0].grid(True, alpha=0.3) axes[0].legend() axes[0].axhline(0, color='k', lw=0.5) # === Efecto del desplazamiento vertical D === for D in [-2, 0, 2, 4]: axes[1].plot(x, np.sin(x) + D, lw=2, label=f'D = {D}') axes[1].axhline(D, ls='--', alpha=0.3) axes[1].set_title('Efecto de D: línea base') axes[1].set_xlabel('x') axes[1].set_ylabel('sin(x) + D') axes[1].grid(True, alpha=0.3) axes[1].legend() plt.show()

Desfase C

Desplaza la onda horizontalmente. C > 0 mueve la onda hacia la derecha, C < 0 hacia la izquierda. En datos temporales, captura "cuándo" ocurre el pico.

Línea base D

Sube o baja toda la onda. Es el valor promedio alrededor del cual oscila la señal. La amplitud A se mide desde D hasta el pico, no desde 0.

Fórmulas prácticas

D = (máx + mín) / 2 (promedio)
A = (máx − mín) / 2 (semi-amplitud)
B = 2π / T (a partir del período)
C = ubicación del primer máximo (para forma factorizada con sin)

Con A, B, C, D podemos describir cualquier oscilación periódica del mundo real: temperatura, mareas, demanda eléctrica, contaminación, ventas estacionales.

Identidades y ecuaciones trigonométricas en SymPy

Verificación simbólica y solución en intervalos

from sympy import * x = symbols('x') # === Verificar identidad pitagórica === expr = sin(x)**2 + cos(x)**2 print(trigsimp(expr)) # → 1 # === Expandir identidad de suma === print(expand_trig(sin(x + pi/3))) # → sin(x)/2 + sqrt(3)*cos(x)/2 # === Ángulo doble === print(expand_trig(cos(2*x))) # → 2*cos(x)**2 - 1 # === Resolver ecuación en [0, 2π) === sol1 = solveset(2*sin(x)**2 - 1, x, domain=Interval(0, 2*pi, False, True)) print(sol1) # → {π/4, 3π/4, 5π/4, 7π/4} # === Factorización con sustitución === # 2cos²(x) + cos(x) - 1 = 0 sol2 = solveset(2*cos(x)**2 + cos(x) - 1, x, domain=Interval(0, 2*pi, False, True)) print(sol2) # → {π/3, π, 5π/3}
Función SymPy Qué hace
trigsimp(expr) Simplifica expresiones trigonométricas
expand_trig(expr) Expande sin(A+B), cos(2x), etc.
solveset(eq, x, domain) Resuelve ecuaciones en un intervalo
Interval(0, 2*pi, False, True) Intervalo [0, 2π) (cerrado-abierto)

Conexión con la teoría

SymPy hace exactamente las mismas manipulaciones simbólicas que hicimos a mano: identidad pitagórica, factorización con sustitución u = cos(x), soluciones generales con +2nπ.

{π/4, 3π/4, 5π/4, 7π/4} {π/3, π, 5π/3}

Aplicación: si modelamos la generación solar como P(t) = A·sin(B(t−C))+D, resolver P(t) = umbral nos dice las horas en que la generación supera ese nivel.

Codificación cíclica de variables temporales

El "problema de medianoche" y cómo resolverlo

import numpy as np import pandas as pd def codificar_ciclico(df, columna, periodo): """Mapea una variable cíclica al círculo unitario.""" df[f'{columna}_sin'] = np.sin( 2 * np.pi * df[columna] / periodo) df[f'{columna}_cos'] = np.cos( 2 * np.pi * df[columna] / periodo) return df # Ejemplo: codificar hora del día df = pd.DataFrame({'hora': range(24)}) df = codificar_ciclico(df, 'hora', 24) # Verificar: hora 23 y hora 0 están CERCA h23 = np.array([df.loc[23, 'hora_sin'], df.loc[23, 'hora_cos']]) h0 = np.array([df.loc[0, 'hora_sin'], df.loc[0, 'hora_cos']]) print(f"Distancia 23→0: {np.linalg.norm(h23-h0):.3f}") # ≈ 0.261 (cerca, como debe ser) # Comparar con codificación cruda print(f"Distancia cruda |23-0|: {abs(23-0)}") # ≈ 23 (incorrecto: parecen muy lejanas) # Períodos comunes: # hora → 24, día semana → 7, # mes → 12, día del año → 365.25

El problema de medianoche

Si codificamos la hora como un número (0, 1, ..., 23), un modelo de ML cree que la hora 23 está a 23 unidades de la hora 0. Pero en realidad están a 1 hora de distancia.

La solución: el círculo unitario

xsin = sin(2π·x/T), xcos = cos(2π·x/T). Cada valor se mapea a un punto único (sin, cos) sobre el círculo unitario.

¿Por qué ambas componentes?

Solo con seno: sin(2π·6/24) = sin(2π·18/24) en algunos casos. Las 6 a.m. y las 6 p.m. quedarían confundidas. El coseno rompe la ambigüedad.

Reducción de dimensionalidad: codificar hora + día + mes + día del año con one-hot necesita 408 columnas. Con sin/cos solo 8 columnas. Reducción del 98%.

Ajuste sinusoidal con curve_fit

Encontrar A, B, C, D automáticamente desde datos reales

import numpy as np import matplotlib.pyplot as plt from scipy.optimize import curve_fit # === Modelo a ajustar === def sinusoide(t, A, B, C, D): return A * np.sin(B * (t - C)) + D # === Datos simulados de demanda CR === np.random.seed(42) t_obs = np.arange(0, 48, 0.5) # 2 días y_real = sinusoide(t_obs, 400, np.pi/12, 6, 1200) y_obs = y_real + np.random.normal(0, 30, len(t_obs)) # === Estimaciones iniciales (CRÍTICO) === A0 = (y_obs.max() - y_obs.min()) / 2 # ≈ 400 D0 = y_obs.mean() # ≈ 1200 B0 = 2 * np.pi / 24 # período conocido C0 = 6 p0 = [A0, B0, C0, D0] # === Ajuste de mínimos cuadrados no lineales === popt, pcov = curve_fit(sinusoide, t_obs, y_obs, p0=p0) A_fit, B_fit, C_fit, D_fit = popt print(f"A={A_fit:.1f}, B={B_fit:.3f}, " f"C={C_fit:.2f}, D={D_fit:.1f}") # === R² (bondad de ajuste) === y_pred = sinusoide(t_obs, *popt) ss_res = np.sum((y_obs - y_pred)**2) ss_tot = np.sum((y_obs - y_obs.mean())**2) r2 = 1 - ss_res/ss_tot print(f"R² = {r2:.4f}")

curve_fit en una línea

Recibe la función del modelo, los datos x, los datos y y unas estimaciones iniciales p0. Devuelve los parámetros óptimos popt y la matriz de covarianza pcov.

Las estimaciones iniciales son críticas

curve_fit usa Levenberg–Marquardt, que puede atascarse en mínimos locales. La estrategia es: D0 = promedio, A0 = (máx−mín)/2, B0 desde el período conocido (o desde la FFT), C0 razonable.

A=400.2, B=0.262, C=6.01, D=1200.1 R² = 0.9876

Calidad del ajuste con R²

El R² mide qué fracción de la varianza de los datos explica el modelo. R² > 0.95 en datos periódicos es excelente. Es la misma métrica que vimos en regresión lineal y polinomial.

Esta es la técnica estándar en ciencia de datos para modelar cualquier señal periódica: mareas, demanda eléctrica, temperatura, ventas estacionales, llegadas turísticas.

Codificación posicional en transformers

Cómo ChatGPT, Claude y Gemini saben la posición de cada palabra

import numpy as np import matplotlib.pyplot as plt def codificacion_posicional(max_len, d_model): """Codificación posicional sinusoidal de Vaswani 2017.""" pe = np.zeros((max_len, d_model)) pos = np.arange(max_len)[:, np.newaxis] div = np.exp(np.arange(0, d_model, 2) * -(np.log(10000) / d_model)) pe[:, 0::2] = np.sin(pos * div) # dim pares pe[:, 1::2] = np.cos(pos * div) # dim impares return pe # Generar y visualizar pe = codificacion_posicional(50, 64) fig, ax = plt.subplots(figsize=(12, 5), layout='constrained') im = ax.imshow(pe, aspect='auto', cmap='RdYlBu') ax.set_xlabel('Dimensión del modelo') ax.set_ylabel('Posición del token') ax.set_title('Positional encoding sinusoidal') fig.colorbar(im, ax=ax) plt.show() # Cada fila es la "huella" única de una posición # Las primeras dimensiones cambian rápido, # las últimas cambian lento

El problema

Los transformers procesan todas las palabras en paralelo. La self-attention es invariante al orden. ¿Cómo le decimos al modelo que "el gato negro" es diferente de "negro gato el"?

La solución: senos y cosenos

PE(pos, 2i) = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d)
Cada dimensión usa una frecuencia distinta. Es como un sistema binario continuo.

Por qué funciona

Por las identidades de suma sin(A+B) y cos(A+B): la posición (pos+k) es una rotación de la posición pos. El modelo aprende a atender por posición relativa, no absoluta.

Esta es la base matemática de RoPE (Rotary Position Embedding), usado por LLaMA, Mistral, Qwen, DeepSeek, Gemma y prácticamente todo LLM moderno. La trigonometría está en el corazón de la IA generativa.

Similaridad coseno y análisis de Fourier

Métricas y técnicas para comparar y descomponer señales

import numpy as np from scipy.fft import rfft, rfftfreq # === Similaridad coseno === def sim_coseno(a, b): return np.dot(a, b) / ( np.linalg.norm(a) * np.linalg.norm(b)) # Perfiles turísticos de provincias CR # [playa, montaña, cultura, aventura, gastro] san_jose = np.array([0.2,0.3,0.9,0.4,0.8]) guanacaste = np.array([0.95,0.2,0.3,0.7,0.5]) cartago = np.array([0.1,0.9,0.7,0.5,0.4]) print(f"SJ-Cartago: {sim_coseno(san_jose,cartago):.3f}") print(f"SJ-Guanacaste: {sim_coseno(san_jose,guanacaste):.3f}") # === FFT: detectar periodicidad === # Simular llegadas turísticas mensuales (10 años) meses = np.arange(120) turismo = (200000 + 1000*meses + 50000*np.sin(2*np.pi*meses/12) + np.random.normal(0, 5000, 120)) fft_vals = rfft(turismo - turismo.mean()) freqs = rfftfreq(len(turismo)) magnitud = np.abs(fft_vals) # Pico en freq = 1/12 → período de 12 meses # confirma estacionalidad anual

Similaridad coseno

cos(θ) = (a·b) / (||a||·||b||). Mide el ángulo entre vectores, no su magnitud. Rango: [−1, 1]. Es la métrica estándar en NLP, búsqueda semántica y RAG.

FFT (transformada rápida de Fourier)

Descompone cualquier señal periódica en sus frecuencias componentes. Si una serie tiene estacionalidad anual, la FFT la detecta como un pico en la frecuencia 1/12 (en datos mensuales).

El teorema de Fourier

Toda señal periódica se puede escribir como suma de senos y cosenos de diferentes frecuencias. Es la base de Prophet (Meta) para modelar estacionalidad.

Para sus proyectos: si sospechan estacionalidad en sus datos, scipy.fft.rfft() la detecta directamente. Los picos del espectro les dicen exactamente cuáles son los períodos.
