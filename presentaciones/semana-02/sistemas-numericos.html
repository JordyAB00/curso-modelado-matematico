El gancho: cómo genera imágenes Stable Diffusion

Una EDO neural transforma ruido en una imagen coherente

La idea matemática

Imaginen una imagen como un punto en un espacio de altísima dimensión. La generación tiene dos direcciones:

  • Forward (entrenamiento): añadir ruido gradualmente a la imagen hasta que sea solo estática. Esto define una EDE forward.
  • Reverse (generación): partir de ruido puro y, paso a paso, "denoiser" hasta recuperar una imagen coherente. Esto resuelve una EDO neural cuyo campo vectorial fue aprendido.

Flow Matching: el avance de 2023 que cambió el campo

Lipman, Chen, Ben-Hamu, Nickel y Le (Meta AI, ICLR 2023, arXiv:2210.02747) propusieron entrenar directamente una red neuronal que aprenda el campo vectorial de una EDO: dx/dt = v_θ(x, t). Stable Diffusion 3, Sora y FLUX usan variantes de esta técnica.

El mensaje para ustedes como futuros ingenieros en ciencia de datos: las ecuaciones diferenciales no son tema antiguo. Son la maquinaria matemática detrás de la IA generativa de 2026.

scipy.integrate: la herramienta del curso

Cómo resolver una EDO sin escribir la fórmula a mano

La función estrella: solve_ivp

SciPy resuelve numéricamente cualquier sistema de EDOs de la forma dy/dt = f(t, y). La documentación oficial v1.17.0 (enero 2026) dice literalmente: "For new code, use scipy.integrate.solve_ivp to solve a differential equation".

Método Tipo Cuándo usarlo
RK45 Runge-Kutta explícito orden 5(4) Por defecto, problemas "típicos" no rígidos
RK23 Bogacki-Shampine orden 3(2) Cuando aceptan tolerancia baja

Regla práctica: empiecen siempre con RK45 (es el predeterminado). Si ven muchos pasos rechazados o advertencias sobre el tamaño del paso, prueben LSODA. Si LSODA conmuta a BDF, su problema es rígido: pasen explícitamente a Radau.

Demo en vivo: el modelo Lotka-Volterra

Depredador y presa en código limpio y moderno

import numpy as np import matplotlib.pyplot as plt from scipy.integrate import solve_ivp def lotka_volterra(t, z, alpha, beta, delta, gamma): x, y = z return [alpha*x - beta*x*y, delta*x*y - gamma*y] params = (1.1, 0.4, 0.1, 0.4) sol = solve_ivp(lotka_volterra, t_span=(0, 50), y0=[10, 5], args=params, method='RK45', rtol=1e-8, atol=1e-10, t_eval=np.linspace(0, 50, 1000)) # sol.t: tiempos | sol.y[0]: presa | sol.y[1]: depredador

t_eval

Define en qué puntos queremos la solución. No tiene nada que ver con la malla interna del solver, que es adaptativa.

rtol y atol

Tolerancias relativa y absoluta. Valores típicos: rtol=1e-6 y atol=1e-9. Bajar estos valores cuando se necesita precisión.

El modelo SIR en código

El sistema que vimos a mano, resuelto numéricamente

def sir(t, y, beta, gamma): S, I, R = y N = S + I + R dS = -beta * S * I / N dI = beta * S * I / N - gamma * I dR = gamma * I return [dS, dI, dR] # Parametros tipicos para gripe estacional beta, gamma = 0.3, 0.1 # R0 = beta/gamma = 3 N = 1000; I0 = 1; S0 = N - I0 sol = solve_ivp(sir, (0, 160), [S0, I0, 0], args=(beta, gamma), t_eval=np.linspace(0, 160, 500), rtol=1e-8)

Estructura general

El campo vectorial recibe (t, y, *args) y devuelve una lista de derivadas. La función puede crecer a 10 o 20 compartimentos sin cambiar la estructura.

R0 explícito

El parámetro R0 = beta/gamma define si la epidemia crece o se extingue. Para SARS-CoV-2 original era ~2.5; para sarampión es ~15.

Detectar eventos durante la simulación

"Avísame cuándo pasa algo importante"

Una funcionalidad de solve_ivp que no tiene odeint: detectar automáticamente cuándo la solución cruza un umbral (pico epidémico, extinción, primer paso por cero).

# Detectar el pico de infectados: cuando dI/dt = 0 def pico_infectados(t, y, beta, gamma): S, I, R = y N = S + I + R return beta * S * I / N - gamma * I # dI/dt pico_infectados.terminal = False # continuar despues del evento pico_infectados.direction = -1 # solo de positivo a negativo sol = solve_ivp(sir, (0, 200), [S0, I0, 0], args=(beta, gamma), events=pico_infectados) print(f"Pico de epidemia en el dia {sol.t_events[0][0]:.1f}") print(f"Infectados en el pico: {sol.y_events[0][0][1]:.0f}")

En producción real, los eventos sirven para "tomar decisión cuando algo pasa": activar una alarma cuando un tanque se vacía, parar una simulación de mercado cuando un precio cruza un umbral, identificar el día del pico para planificación hospitalaria.

El modelo SEICR aplicado a COVID-19 en Costa Rica

En mayo de 2020, el doctor Tomás de-Camino-Beck (Automata Innovations y Lead University) publicó en medRxiv un modelo SEIR modificado con un compartimento C de "Confinados", específicamente para Costa Rica.

La innovación clave

Las intervenciones no farmacéuticas (confinamiento, distanciamiento) se modelan como un flujo dinámico entre S y C. Al confinar gente, se reduce efectivamente la población susceptible disponible para contagio.

El sistema

S → E → I → R con un flujo paralelo S ↔ C. Los parámetros se calibran a la curva real de casos publicada por el Ministerio de Salud durante marzo y abril de 2020.

Calibración a datos reales

Ajustar parámetros con mínimos cuadrados robustos

from scipy.optimize import least_squares # 1. Generar datos sinteticos con parametros reales conocidos N, I0 = 1000, 1 beta_real, gamma_real = 0.30, 0.10 # R0 real = 3.0 t_data = np.linspace(0, 100, 50) sol_real = solve_ivp(sir, (0, 100), [N-I0, I0, 0], args=(beta_real, gamma_real), t_eval=t_data, rtol=1e-8) rng = np.random.default_rng(42) I_data = sol_real.y[1] + rng.normal(0, 8, len(t_data)) # 2. Definir los residuos y ajustar def residuos(theta): beta, gamma = theta sol = solve_ivp(sir, (0, 100), [N-I0, I0, 0], args=(beta, gamma), t_eval=t_data, rtol=1e-8) return sol.y[1] - I_data res = least_squares(residuos, [0.5, 0.05], bounds=([0, 0], [5, 5]), loss='soft_l1') # robustez ante outliers beta_hat, gamma_hat = res.x print(f"R0 real: {beta_real/gamma_real:.2f}") print(f"R0 estimado: {beta_hat/gamma_hat:.2f}")

loss='soft_l1' o 'huber'

Usar pérdidas robustas (no cuadráticas) cuando los datos epidemiológicos tienen semanas de sub-reporte u outliers. Es una de las "buenas prácticas modernas" que diferencia un modelo de juguete de uno utilizable en producción.

Calibración bayesiana: el estándar de 2026

No un valor estimado, una distribución completa

Mínimos cuadrados da un valor puntual. La inferencia bayesiana da una distribución completa: "no sé exactamente cuánto es R0, pero con 95 % de probabilidad está entre 2,1 y 2,8". Esto es lo que se usa hoy en epidemiología profesional, finanzas cuantitativas y ML aplicado.

import pymc as pm from pymc.ode import DifferentialEquation # Reutilizamos los datos sinteticos (t_data, I_data, N, I0) del slide anterior # El SIR como funcion compatible con pymc.ode def sir_ode(y, t, p): S, I, R = y[0], y[1], y[2] beta, gamma = p[0], p[1] return [-beta*S*I/N, beta*S*I/N - gamma*I, gamma*I] sir_model = DifferentialEquation(func=sir_ode, times=t_data, n_states=3, n_theta=2) with pm.Model() as modelo_sir: # Priors: lo que sabemos antes de ver los datos beta = pm.LogNormal("beta", mu=np.log(0.3), sigma=0.5) gamma = pm.LogNormal("gamma", mu=np.log(0.1), sigma=0.5) # Integrar la EDO dentro del modelo bayesiano sol = sir_model(y0=[N-I0, I0, 0], theta=[beta, gamma]) # Verosimilitud sobre los infectados observados pm.NegativeBinomial("obs", mu=sol[:, 1], alpha=10, observed=I_data) idata = pm.sample(2000, tune=2000, target_accept=0.95) # Distribucion posterior completa de R0 = beta/gamma R0_post = idata.posterior["beta"] / idata.posterior["gamma"]

Más allá de SciPy: el ecosistema diferenciable

Cuando el solver es una capa de una red neuronal

SciPy resuelve EDOs. Lo que cambió la industria fue poder propagar gradientes a través del solver. Eso significa que los parámetros de un modelo de EDOs se aprenden por descenso de gradiente, igual que los pesos de una red neuronal.

Diffrax (JAX)

Patrick Kidger, versión 0.7.2 (febrero 2026). Solver unificado para EDOs/EDEs/CDEs en JAX, nativo en GPU, diferenciable. Es el solver de referencia cuando se necesita JIT más GPU más diferenciabilidad. Usado en biología de sistemas, control óptimo, modelado farmacocinético.

torchdiffeq (PyTorch)

Ricky Chen, implementación de referencia de los Neural ODEs (NeurIPS 2018, Best Paper). Provee odeint y odeint_adjoint con gradientes O(1) en memoria. Caballo de batalla en PyTorch para EDOs neuronales.

DifferentialEquations.jl (Julia)

Chris Rackauckas. El ecosistema técnicamente más maduro y rápido del mundo para EDOs. Mencionado para que sepan que existe, no es nuestro stack del curso.

NumPyro + Diffrax

Para Bayesian SIR/SEIR en GPU, la combinación más performante de 2026. Alternativa moderna a PyMC para casos donde el modelo es muy grande o se necesita aceleración.

PINNs: redes neuronales informadas por física

Una nota honesta sobre dónde están en 2026

La promesa de los PINNs

Una red neuronal que aprende la solución de una ecuación diferencial parcial directamente, sin necesidad de mallas tradicionales. NVIDIA mantiene PhysicsNeMo como framework comercial; SAP los listó como tema de IA estratégica para 2025.

Dónde sí funcionan

Problemas inversos (estimar coeficientes desde datos), geometrías complejas sin malla, escasez extrema de datos donde la física es conocida parcialmente.

Donde no funcionan (todavía)

Problemas multi-escala, dinámicas de alta frecuencia, problemas directos estándar donde CFD o FEM siguen ganando por precisión y velocidad.

Conclusión equilibrada: PINNs son área activa de investigación con resultados mixtos. Reviews recientes (arXiv:2411.18240, ScienceDirect 2025) documentan limitaciones serias. No las descartemos, pero tampoco las traten como solución mágica. En este curso solo las mencionamos.

Validación correcta para EDOs ajustadas a series temporales

Lo que no se aprende en un curso estándar de ML

Por qué k-fold no funciona aquí

La validación cruzada estándar (k-fold) baraja los datos aleatoriamente. En series temporales eso rompe la dependencia temporal: el modelo "ve el futuro" durante el entrenamiento.

Hold-out hacia adelante

Entrenar con datos hasta el momento t. Predecir t+h. Repetir incrementalmente.

Backtesting epidemiológico

Ajustar hasta SE k, predecir SE k+1 a k+4. Repetir desplazando una semana.

WIS (Weighted Interval Score)

Métrica estándar en los Forecast Hubs del CDC y otros organismos.

Análisis de sensibilidad: SALib

Herramienta canónica en Python para identificar qué parámetros importan más en la salida del modelo. Métodos Morris (screening barato) y Sobol (descomposición de varianza global). Es lo que distingue un modelo "que ajusta" de un modelo "que se entiende".

Limitaciones de los modelos compartimentales

Lo que un modelador serio debe poder enumerar

1. Asumen mezcla homogénea

Todos pueden contagiar a todos por igual. Realidad: las redes de contacto son altamente heterogéneas.

2. Población cerrada

No hay nacimientos, muertes ni migración a menos que se añadan explícitamente.

3. Parámetros constantes

Beta no cambia con intervenciones, estaciones o comportamiento. En la realidad sí cambia.

4. Determinismo

A poblaciones pequeñas la estocasticidad importa. Se usan ecuaciones diferenciales estocásticas (EDE).

5. Identificabilidad

Distintos pares (beta, gamma) pueden producir curvas similares, especialmente con datos truncados o incompletos. Solucionarlo requiere priors informativos en el enfoque bayesiano.

Un modelador profesional no se enamora del modelo. Enumera sus limitaciones primero, propone mejoras después. Esta es la diferencia entre el ejercicio académico y el modelado para la toma de decisiones reales.
